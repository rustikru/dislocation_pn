# Доработка: кнопка «Добавить блок подписантов» в форме акта

**Дата:** 2026-07-24
**Автор задачи:** BekmansurovRR
**Область:** только папка `gu23`

## Суть

Сейчас количество подписантов у акта фиксировано: оно задаётся статическим
конфигом `signerRules` в `gu23/js/form.js` (для каждого типа акта — своё число
слотов «Работник предприятия» / «Работник ОАО РЖД»). Нужно добавить кнопку,
которая позволяет добавлять дополнительные блоки подписантов вручную, с выбором
типа: **«Работник предприятия»** (`own`) или **«Работник ОАО РЖД»** (`rzd`).

## Главный вывод

Доработка **фронтовая — только `gu23/js/form.js`**. PHP и пакет PL/SQL
менять НЕ нужно.

### Почему бэкенд не трогаем (проверено по коду)

Количество подписантов нигде в БД не зашито — оно выводится из строк, которые
присылает фронт:

- `gu23_save_act` парсит `p_signers` в цикле `while` и вставляет каждого через
  `insert_signer` (со `stype` = `own`/`rzd`) — сколько прислали, столько и создаст;
- `gu23_approval_init` / `gu23_approval_get_signers` / `sync_act_status` /
  `gu23_approval_next_signer` работают по `where stype = 'own'` — динамически,
  без фиксированного числа;
- PHP просто пакует массив: `packRows($signers, ['id','fio','post','org','stype'])`.

Единственное «фиксированное количество» — фронтовый конфиг `signerRules` /
`getSignerSlots` в `form.js`. Значит, вся задача — фронтовая.

## Что и где менять в `gu23/js/form.js`

Опорные места в текущем коде:
- `signerRules` / `getSignerSlots` — конфиг слотов (~стр. 921–966);
- `showSignersFields()` — отрисовка блока подписантов (~стр. 968);
- `putSignersToSlots()` — раскладка сохранённых подписантов по слотам (~стр. 991);
- `showSignerSlot()` — отрисовка одного слота (~стр. 1059);
- `checkRequiredSigners()` — валидация обязательных (~стр. 1311);
- сохранение: `signers: JSON.stringify(activeDraft.signers.filter(Boolean))` (~стр. 1395).

Весь рендер и сбор уже работают **по индексу слота** — поэтому достаточно
сделать список слотов «расширяемым».

### Шаг 1. Хранить добавленные блоки в черновике

Завести поле `activeDraft.extraSlots = []` — массив вида `[{type:'own'}, {type:'rzd'}]`.

### Шаг 2. В `showSignersFields()` склеить базовые и добавленные слоты

```js
const baseSlots  = getSignerSlots(activeDraft.type)
const extraSlots = (activeDraft.extraSlots || []).map(s => ({
  type: s.type,
  required: false,
  helpText: s.type === 'rzd' ? 'Работник станции ОАО «РЖД»' : 'Работник предприятия',
  removable: true,            // чтобы показать кнопку удаления «×»
}))
const signerSlots = baseSlots.concat(extraSlots)
```

Дальше по коду использовать `signerSlots` как раньше (в `putSignersToSlots`,
в цикле рендера, в `showSignerSlot`). В этих функциях менять ничего не нужно —
они уже индексные.

### Шаг 3. Кнопки добавления

В конце `showSignersFields()` (после цикла отрисовки слотов) дописать:

```js
$container.append(`
  <div style="display:flex;gap:6px;margin-top:8px">
    <button type="button" class="btn sm" id="add-signer-own">+ Работник предприятия</button>
    <button type="button" class="btn sm" id="add-signer-rzd">+ Работник ОАО «РЖД»</button>
  </div>`)

$('#add-signer-own').on('click', () => {
  activeDraft.extraSlots.push({ type: 'own' })
  showSignersFields()
})
$('#add-signer-rzd').on('click', () => {
  activeDraft.extraSlots.push({ type: 'rzd' })
  showSignersFields()
})
```

### Шаг 4. Кнопка удаления добавленного блока

В `showSignerSlot()` для слотов с `slotCfg.removable` вывести «×». По клику —
вырезать слот и сдвинуть подписантов:

```js
// i — позиция слота = индекс в activeDraft.signers
const extraIdx = i - getSignerSlots(activeDraft.type).length   // индекс внутри extraSlots
activeDraft.extraSlots.splice(extraIdx, 1)
activeDraft.signers.splice(i, 1)
showSignersFields()
```

## Нюанс: редактирование существующего акта

При открытии акта в `activeDraft.signers` может прийти БОЛЬШЕ подписантов, чем
базовых слотов (если их добавляли ранее). Чтобы «лишние» отрисовались, при
загрузке черновика (до первого вызова `showSignersFields()`) восстановить
`extraSlots` из хвоста массива:

```js
const base = getSignerSlots(activeDraft.type).length
activeDraft.extraSlots = (activeDraft.signers || [])
  .slice(base)
  .map(s => ({ type: s && s.stype === 'rzd' ? 'rzd' : 'own' }))
```

## Что НЕ меняем

- **Валидацию** `checkRequiredSigners` — добавленные слоты `required:false`,
  обязательность базовых не меняется.
- **Сохранение/упаковку** — `activeDraft.signers.filter(Boolean)` уже шлёт
  любой размер массива.
- **PHP и PL/SQL** — не трогаем вообще.

## Внутри блока (без изменений)

У добавленного блока пользователь выбирает тип при добавлении
(«Работник предприятия» = `own` / «Работник ОАО РЖД» = `rzd`), а внутри блока
остаётся уже готовое переключение **«Из справочника / Вручную»** — оно работает
для любого слота по индексу.
