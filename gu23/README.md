# ГУ-23 · Акты общей формы

Модуль формирования и учёта актов общей формы ГУ-23 (простой вагонов).
Подход — как у модуля `shift_change`: страница (`index.php` + `gu23.js` + `gu23.css`),
обращения через `/data.php` → `router.php` → `GuActRepository`. Вся работа с БД —
только через функции пакета `xx_dislocation.gu23_*`.

## Что реализовано
- Создание акта: **начало простоя**, **окончание простоя**, **прочий акт**
- Авторасчёт длительности простоя (дни/часы, всего часов, кал. дни)
- Циклы простоя по вагону: запрет дубля открытого начала, закрытие начала при заведении окончания
- Архив: фильтры (тип, статус, цех) + поиск (номер акта / вагон / причина)
- Карточка акта: реквизиты, вагоны, подписанты, приложения, история
- Приложения: загрузка/удаление/скачивание файлов
- Поиск по вагону
- Кнопка «Запросить в Oracle BI / Дислокация» (пока — имитация в пакете, `select ... from dual`)

Подписание (workflow), печатная форма Word/PDF, отчёты — **вне текущего этапа**.

## Статусы акта
`Черновик` → `Действующий` → (`Закрыт` для акта начала, по которому заведено окончание) · `Аннулирован`

## Локальный запуск (macOS + Oracle в Docker)

1. **Конфиг БД** (один раз):
   ```bash
   cp db_config.local.php.example db_config.local.php
   # отредактируйте $pwd под свой Oracle
   ```
   `connection.php` автоматически использует `db_config.local.php`, если он есть.
   Файл в `.gitignore` — на прод не попадает. Он же включает **dev-обход авторизации**.

2. **Установка в БД** — три скрипта (Run Script / F5) под `XX_ETW`:
   ```sql
   @install_gu23_all.sql   -- 1) структура: таблицы, sequences, представление, типы
   @compile_gu23_pkg.sql   -- 2) пакет (спецификация + тело)
   @fill_gu23_data.sql     -- 3) справочники + dev-пользователь
   ```
   `install_gu23_all.sql` дропает таблицы (полный сброс структуры).
   `compile_gu23_pkg.sql` и `fill_gu23_data.sql` таблицы/данные не дропают —
   их можно гонять отдельно (обновить код пакета / восстановить справочники).

3. **Запуск веб-сервера** из корня проекта `dislocation_pn`:
   ```bash
   DYLD_LIBRARY_PATH=/opt/oracle/instantclient_23_26 php -S localhost:8080 -t .
   ```

4. Открыть: **http://localhost:8080/gu23/**

## Состав модуля
| Файл | Назначение |
|---|---|
| `index.php` | каркас страницы (jQuery 3.7.1 только здесь) |
| `gu23.js` | вся клиентская логика (jQuery + AJAX) |
| `gu23.css` | стили (из утверждённого прототипа) |
| `GuActRepository.php` | серверный репозиторий (роутер-паттерн) |
| `get_file.php` | отдача приложений на скачивание/просмотр |
| `install_gu23_all.sql` | обычные объекты: таблицы, sequences, представление, типы (с дропом) |
| `compile_gu23_pkg.sql` | пакет `xx_disl_gu23_pkg` — спецификация + тело (без дропа таблиц) |
| `fill_gu23_data.sql` | наполнение справочников + dev-пользователь (идемпотентно) |

## БД (префикс `xx_disl_gu23_`, у каждой таблицы свой sequence)
`act`, `act_row`, `file`, `signer`, `hist`, `users`, `counter` (нумерация),
справочники `ref_cex / ref_reason / ref_station / ref_owner / ref_wagon_kind / ref_signer`.

Передача коллекций (вагоны/подписанты) из PHP в пакет — без JSON (совместимо с Oracle 11):
записи — `CHR(30)`, поля — `CHR(31)`.

## Перенос на ПРОД
Локально пакет `xx_dislocation` создаётся в мини-версии (только `gu23_*`).
На проде функции `gu23_*` из `compile_gu23_pkg.sql` нужно **врезать в настоящий
пакет `xx_dislocation`** (он уже существует). Таблицы/типы/sequences создаются из
`install_gu23_all.sql`, справочники — из `fill_gu23_data.sql`.

### Интеграция с реальным источником вагонов
Заменить тело функции `gu23_get_wagon_info` (блок «заглушка») на реальный запрос к
Oracle BI / внутренней дислокации. Поля результата: `wagon_no, owner, kind, st_from,
st_to, cargo, weight, found` (1/0). Условие — станция операции «Углеуральская».
