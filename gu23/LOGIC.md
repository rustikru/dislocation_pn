# ГУ-23 · Логика по типам актов

Документ фиксирует бизнес-логику, зашитую в модуль. Источник правды —
функция `gu23_save_act` (и `gu23_annul_act`) в `compile_gu23_pkg.sql`
плюс клиентская часть в `gu23.js`.

## Общие правила (для всех типов)

- **Типы акта:** `start` (Начало простоя), `end` (Окончание простоя), `other` (Прочий акт).
- **Статусы:** `draft` (Черновик) → `active` (Действующий) → `closed` (Закрыт, только для акта начала) · `annulled` (Аннулирован).
- **Черновик (`draft`)** сохраняется как угодно — обязательные поля НЕ проверяются
  (ни на фронте, ни в пакете). Все проверки ниже срабатывают только при сохранении
  «на подписание» (`active`).
- **Номер акта** (`ГУ23-ЦЕХ-ГОД-000001`) генерируется при наличии цеха. Если черновик
  завели без цеха — номер присваивается позже, когда цех появится.
- **Редактирование:** менять можно ТОЛЬКО черновик. Действующий/закрытый/аннулированный
  акт не редактируется (пакет вернёт ошибку) — доступно лишь **аннулирование** с указанием причины.
- **Дата/время:** ввод коротким форматом (`ддммгг ччмм`) → приводится к `dd.mm.yyyy HH:MM`.
  Преобразования строковые, без смещения часового пояса: что ввёл пользователь — то и в БД.
- **История:** каждое сохранение и закрытие цикла пишется в `xx_disl_gu23_hist`.
- **Подписанты:** при отправке требуется 3 для `start`/`end` и 2 для `other`.

---

## Тип `start` — Начало простоя

Фиксирует момент, когда вагоны встали в простой.

**Поля:** цех, станция, причина, обстоятельства, **дата начала**, список вагонов.

**Источник данных по вагонам:** кнопка **«Запросить данные из Дислокации»**
(`gu23_get_wagon_info`, сейчас заглушка) — подтягивает собственника, род, станции, груз, вес.
Кнопка доступна **только** для этого типа.

**Проверки при отправке (`active`):**
- указан цех;
- указана дата начала;
- добавлен хотя бы один вагон;
- **запрет дубля открытого цикла:** нельзя завести «Начало» по вагону, по которому уже
  есть другой действующий акт начала (вагон уже в открытом простое). Сообщение указывает
  номер мешающего акта. Обход — флаг `force='Y'` (админ-подтверждение в модалке).

**Жизненный цикл:** `active` → становится `closed` автоматически, когда **все** его вагоны
закрыты действующими актами окончания (см. тип `end`).

---

## Тип `end` — Окончание простоя

Закрывает ранее открытый акт начала (полностью или частично).

**Привязка:** обязательно выбирается **открытый акт начала** (`linked_start_id`).
При выборе из него **наследуются** данные: вагоны, причина, цех, станция, дата начала —
поэтому запроса в Дислокацию для этого типа нет (кнопка скрыта).

**Поиск акта начала:** кнопка **«Найти открытый простой»** по введённым номерам вагонов.
Если введённые вагоны принадлежат **разным** открытым актам — выдаётся ошибка со списком
номеров (закрыть разные акты одним окончанием нельзя). Если все из одного акта — он подставляется.

**Проверки при отправке (`active`):**
- выбран открытый акт начала;
- указана дата окончания;
- дата окончания **не меньше** даты начала и **не в будущем**;
- по каждому вагону:
  - **(а)** вагон должен принадлежать выбранному акту начала;
  - **(б)** вагон ещё не закрыт другим действующим актом окончания.

**Расчёт длительности** (дни, часы, всего часов, календарные дни) считается
автоматически по датам начала/окончания.

**Закрытие цикла акта начала:**
- считается «закрытых» вагонов = distinct вагоны по всем действующим окончаниям этого начала;
- если закрыты **все** вагоны начала → акт начала переводится в `closed`,
  в историю: «Цикл простоя полностью закрыт…»;
- иначе → начало остаётся `active`, в историю: «Частично закрыто… (N из M)».

---

## Тип `other` — Прочий акт

Акт общей формы, не связанный с циклом простоя (разовый документ).

- **Нет** дат начала/окончания и расчёта длительности.
- **Нет** привязки к акту начала и логики закрытия циклов.
- **Нет** проверок дубля/принадлежности вагонов.
- **Нет** кнопки «Запросить данные из Дислокации».
- При отправке требуется **2** подписанта (для `start`/`end` — 3).

---

## Аннулирование (`gu23_annul_act`)

- Любой действующий/закрытый акт можно аннулировать с указанием причины → статус `annulled`.
- **Если аннулируется акт окончания** — связанный акт начала, если он был `closed`,
  снова открывается (`active`): цикл считается незакрытым.
